1. Windows NT 4.0

--- Платформа разработки ---

1. Стандартная библиотека языка C/C++
2. Microsoft Platform SDK
3. API Win32, Shell
4. Microsoft Visual Studio 2005 - для разработки,
   Microsoft Visual Studio 6 - для публикации



--- Общие характеристики и функции ---

Харктеристики и функции которые есть у других программ

1. Двух панельный файловый менеджер
2. Операции с файлами: просмотр, копирование, перемещение, удаление,
   выполнение, поиск, сравнение, разбиение 
3. Операции с архивами: просмотр, запаковать, распаковать, изменить
4. FTP клиент
5. Отображение файлов разного типа своим цветом
6. Журнал операций
7. Упорядочивание списка файлов по имени, дате, размеру, типу и другим свойствам
8. Экспорт списка файлов в текст, с рекурсией директорий
9. Двух цветная подсветка фона строк панелей
10. Drug & Drop
11. Горячие клавиши операций
12. Отображение контекстного меню оболочки



--- Особые характеристики и функции ---

Харктеристики и функции которые есть только у Fileman

1. Упрощённый итерфейс
2. Раздельная работа панелей
3. Проcмотр CD-Text (Отображение названий треков AudioCD, вместо строчки "Track XX")
4. Изменение иконки ярлыка при запуске, если кнопка на панели задач не отображается 
5. Ширина колонок задаётся относительно размера главного окна и изменяется пропорцианально
6. Прозрачное копирование wave данных с AudioCD



--- Конфигурационный файл ---

Все сохраняемые параметры записываются в файл fileman.ini



--- Формат сохранения парметров элементов диалога. Файл fileman.ini ---

Название секции - идентификатор элемета диалога с названием типа, определены в resource.h
Название поля   - мнемоническое название параметра, определены в процедуре saveDlgItemProperties
Значение поля   - значение параметра

Пример:

[Название секции]
Название поля=Значение поля



--- Сохраняемые параметры элеметов диалога ---

[Тип элемента]                  [Сохраняемый параметр]
ComboBox                        Выбранная позиция (CB_GETCURSEL/CB_SETCURSEL)
EditControl                     Отображаемый текст (WM_GETTEXT/WM_SETTEXT)
DialogBox                       Структура WINDOWPLACEMENT (структура из Windows SDK)
ListView                        Относительные ширины колонок




--- Сохраняемые параметры программы ---

1. Соответствие цветов типам файлов.
   Название секции: FileTypesColors
   Формат поля: Тип файла=Цвет
   Тип файла - расширение в имени файла, например exe
   Цвет - значение цвета в формате RGB (структура из Windows SDK)
2. Индекс колонки по которой упорядочены строки панели
3. Порядок упорядочивания по возрастанию или убыванию




--- Требования к оформлению кода ---

1. Для именования использовать camelNotation
2. Если в блоке количество операторов больше одного, то блок обрамляется скобками
3. Парные скобки блоков распалогать на отдельной строке и на одной вертикальной линии
4. Блок операторов if, switch и case распологать с новой строки
5. Допускается запись блока на одной строке с операторами if, switch и case
   если блок состоит из менее двух операторов
6. Названия процедур начинаются с маленькой буквы и состоят не менее чем из одного слова.
   Первое слово в названии процедуры глагол.

   

   
--- Названия свойств элементов ---

Названия свойств элементов используются при вызове
WinAPI-процедур SetProp и GetProp

* Панель со списком файлов
  "Counted item" - индекс строки, у которой подсчитывается размер файлов
  "Props" - указатель на структору типа FmPanelProps, содержит свойства панели

* Диалог операции "копирование"
  hThread - дескриптор потока 
  isSuspended - флаг паузы потока. Устанавливается в TRUE, если поток остановлен


Дескрипторы потоков операций "Подсчёт" и "Листание"
сохраняются в свойствах панели



--- Использование структуры WIN32_FIND_DATA ---

Указатель на экземпляр структуры WIN32_FIND_DATA сохраняется в LVITEM::Param.
Память для структуры выделяется динамически LocalAlloc.

DWORD dwReserved0 - цвет шрифта при отображении в List View;
